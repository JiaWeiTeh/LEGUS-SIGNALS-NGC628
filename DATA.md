# Data files

Large / raw data is **not** stored in this Git repo (it exceeds GitHub's file limits).
It lives locally in your working copy and is git-ignored. This file records what each
piece is and where it belongs, so the pipeline can be reproduced on another machine.

## Kept in Git (small inputs)

- `lib/LEGUS/Catalog.csv` — LEGUS NGC 628 star-cluster catalogue
  (columns described in `lib/LEGUS/LEGUS_description.txt`).
- `lib/SIGNALS/NGC628_catalog_WCS_corr.fits` — SIGNALS Hα H II-region catalogue
  (columns described in `lib/SIGNALS/SIGNALS_description.txt`).

## Kept local only (git-ignored)

- `lib/SLUG2/` — SLUG2 model libraries:
  - `cluster_slug/*.tab` — Bayesian PDF tables (q0 / mass / age / av) per stellar track
    (`modc020` = Geneva, `modp020` = Padova).
  - `cluster_lib/*.fits`, `*summary.txt` — SLUG2 cluster library. The `1e5_SC_*` fits
    are used by `fg4` / `fg9`; the `1e7_SC_z0_*` fits are not referenced by any code.
  Obtain from the SLUG2 project / your own SLUG2 runs and place under `lib/SLUG2/`.
- `src/dat/*.npy` — pipeline-generated arrays (combined catalogue, Monte-Carlo Q(H⁰)
  distributions, clusterslug tables, medians, budgets). **Regenerated by the code**
  (`src/tools/create_*` and the `src/plots/fg*` scripts); not versioned.
- `src/figs/`, other `*.pdf` — generated figures.

## Regenerating the derived data

`combined_catalogue.npy` needs only the bundled LEGUS + SIGNALS catalogues — no SLUG2
data. Its builder has no command-line entry point, so call the function from the repo
root:

```
python -c "from src.tools.create_combined_table import create_combined_table as f; f()"
```

The clusterslug PDF tables and the heavy Monte-Carlo `*LHa_QH0_noCons*.npy` arrays
additionally require the SLUG2 libraries under `lib/SLUG2/`; the latter are produced by
the `src/plots/fg*` scripts themselves (e.g. `fg5_LHa_vs_QH0.py` with its `reRun` flag),
not by `src/tools/`. All `*.npy` outputs land in `src/dat/`.
